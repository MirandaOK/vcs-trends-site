Introduction

Version control systems (VCS) are essential tools that help developers manage changes, track modification in files over time, revert changes, collaborate with others and merge their work.

Key features

  1. Repository - The database of fiels, folders and the history
  2. Commit(Check-in) - Records a set of changes to the repository. Commits usually include a message describing the change
  3. Update (Pull/Checkout) - Retriving the latest changes from the repository into your project
  4. Branch - A parallel line of development â€“ a pointer to a series of commits separate from other lines. Branches allow working on features or fixes in isolation
  5. Merge - Combines changes from one branch into another, used to integrate work of different people
  6. Conflict - When two incompatible changes are made to the same part of a file, a merge conflict occurs
  7. Tag/release - Marking a specific, significant commit
  8. Revert - Undoing a change if it introduces any problems
