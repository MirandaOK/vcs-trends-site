History of VCS

Introduction

Version control systems (VCS) are essential tools that help track changes in files (usually source code) over time. They allow multiple people to collaborate on projects, keep a history of modifications, and revert to earlier versions when needed.

Important Dates
  1. 1972: Source Code Control System (SCCS)
  2. 1982: Revision Control System (RCS)
  3. 1986: Concurrent Versions System (CVS)
  4. 2000: Subversion (SVN)
  5. 2005: Git and Mercurial
  6. 2008: GitHub launched
  7. 2010s: Wide adoption of Git and migration from older systems
  8. Late 2010s: GitOps and modern practices
  9. 2020s: AI and large-scale collaboration

Version control systems can be centralized or distributed:

Centralized Version Control (CVCS) - centralized tools are for example RCS, CVS and Subversion.

Distributed Version Control (DVCS) - distributed tools are for example Git or Mercurial.

Use of the tools:

  1. RCS (Revision Control System) - mostly small projects
  2. CVS (Concurrent Versions System) - small to medium team projects

  4. Git - any scale projects
