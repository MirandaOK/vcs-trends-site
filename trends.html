GitOps

The idea is to treat infrastructure configuration and deployment scripts as code in a Git repository and to use Git workflows (commits, pull requests) to manage changes to those configurations.

Principles of automation and consistency:

  1. Single Source of Truth
  2. Pull Requests for Changes
  3. Continuous Deployment Automation

Integration with CI/CD Pipelines

  • Continuous Integration (CI) - Automatic compiling and running tests to catch any issues quickly
  • Continuous Delivery/Deployment (CD) - Delivering changes to staging and production environments and automatic deployment of commits that pasted earlier tests

CI/CD ensures each change is easy to track, change and deploy and because of that leads to a faster release. CI/CD notifies when something is wrong, so developers can fix it more easily and quickly. Because of that, the overall code quality is much better.

Hosting repository platforms

Benefitial functions:

  • Built-in CI services
  • Social coding
  • Access Control and Integrations

The most popular platforms:

  1. GitHub
  2. GitLab
  3. Bitbucket

Role of AI

Nowadays, AI can provide valuable feedback such as pointing out issue in a pull request and advising how to fix them. This way humans can focus on looking for more complicated errors while the simple ones can be caught automatically.
